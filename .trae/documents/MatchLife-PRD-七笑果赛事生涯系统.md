# MatchLife - 七笑果赛事生涯系统 PRD

## 1. 产品概述 (Product Overview)
**MatchLife** 是一个专注记录运动员比赛生涯成绩的小系统。系统从羽毛球U系列比赛起步，未来将涵盖各类专业、业余赛事。不仅记录历史比赛成绩，还能提供当前正在进行中比赛的数据实时检索与排名统计。
**产品愿景**：为体育行业用户提供最便捷、最友好的比赛数据检索与生涯记录工具，解决手动收集比赛数据的繁琐问题，让运动员轻松追踪自己的比赛生涯。

## 2. 目标用户 (Target Users)
- **运动员**：追踪自己的比赛生涯、历史成绩、胜率与表现趋势。
- **教练**：分析队员及对手的比赛数据，制定训练与比赛策略。
- **家长**：关注孩子的比赛进展与成长轨迹。
- **协会等组织**：获取宏观的赛事数据统计、区域排名与活跃度分析。

## 3. 核心功能 (Core Features)
系统以“搜索”为核心，保持极致简洁。
1. **运动员信息检索**：快速查询运动员基本信息及过往参赛概况。
2. **赛事检索**：查询历史赛事及当前正在进行中的实时赛况。
3. **运动员比赛成绩检索**：查看具体运动员在某项赛事中的详细成绩与对阵比分。
4. **数据统计与排名**：提供赛事维度的排行榜、胜率统计及多维度数据看板。

## 4. 界面与交互体验 (UI & UX)
- **极致简洁**：用户打开页面后，视觉中心即为一个**全局大搜索框**。
- **灵活检索**：支持**精准查询**与**模糊查询**。
- **多维度检索条件**：搜索框及筛选面板支持按以下条件组合查询：赛事名称、运动员名称、赛事日期、赛事类型（如专业、业余、U系列等）。
- **七笑果 UI 规范**：
  - 主色调：品牌暖橙色系渐变（亲和、运动、温暖）。
  - 字体与元素：圆润饱满的几何元素与字体，穿插明黄色虚线。
  - 响应式：桌面优先设计，完美适配移动端（触摸区域 > 44px），保障微信内置浏览器体验。

## 5. 阶段性实现计划 (Phased Implementation)

### 第一阶段（当前阶段）：U系列轮询系统与 Web 端免登录查询
**核心定位**：依托微信生态（“七笑果-文体有料”公众号），无需登录，从 ymq 平台自动轮询拉取 U 系列比赛数据并入库，提供快速搜索、筛选与基础统计。

#### 5.1 页面规划 (最小可用版本 MVP)
1. **首页（智能搜索与概览）**：
   - 居中突出的全局搜索框（支持选手/赛事/比赛编号模糊搜索，实时联想）。
   - 高级筛选器（折叠式）：按日期、U组别、赛事、胜负方过滤。
   - 结果列表：卡片式比赛记录（时间、赛事、双方、比分、胜者）。
   - 概览数据与同步状态：展示当前库中总比赛数、最近更新时间及简要同步状态。
2. **比赛详情页**：
   - 单场比赛的完整信息（赛事、组别、时间地点、对阵双方、比分/局分、来源链接）。
   - 上下文跳转：提供“同赛事更多比赛”、“该选手近期比赛”快捷入口。
3. **选手生涯页**：
   - 选手档案：关键指标（总参赛、胜率、最近活跃）。
   - 历史时间轴：按时间线展示比赛结果与颜色编码（胜/负）。
   - 表现分析：胜率趋势、对手对阵分布。
4. **统计与监控看板页**：
   - 赛事统计：U系列赛事核心指标（总场次、总人数、地区分布、胜率排行）。
   - 监控看板：显示 ymq 平台数据同步状态（最近成功/失败时间、拉取条数）。
   - 数据导出：一键导出 CSV 格式明细与报表。

#### 5.2 核心流程与数据轮询策略
- **用户动线**：
  首页智能搜索 -> 点击进入比赛详情 -> 点击选手名称进入生涯页 -> 点击顶部导航查看全局赛事统计。
- **轮询策略 (从 ymq.me 获取)**：
  - **频率**：每 6 小时轮询一次，避免对源平台造成压力。
  - **范围**：仅获取 U 系列比赛数据。
  - **更新机制**：基于时间戳和 `ymq_match_id` 增量 UPSERT 去重。
  - **异常处理**：失败自动重试 3 次，记录详细日志并体现于监控状态。

### 第二阶段：用户体系与 SSO 接入
- **身份认证**：接入 SSO（单点登录）机制。
- **个性化功能**：提供认领运动员身份、关注特定运动员、生成专属生涯海报等个性化增值服务。

## 6. 成本与运维建议
- **最小化基础设施**：使用 Supabase 免费额度（Database + Edge Functions）控制调用与存储成本。
- **数据清理与归档**：仅存储搜索与统计必需字段，超过 2 年的非核心明细数据可归档。
- **缓存策略**：搜索结果缓存 1 小时，统计看板预计算，减少数据库直接查询压力。
